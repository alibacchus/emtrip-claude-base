「未確定要素が多すぎて話せない／何を話するの？」というブロックが来たら、以下の構造で切り返すこと：

**コア反転**：
> 「確証ある事実の共有ならメール／スプシで十分。
>  確証がない事象を各々がどう解釈するかを擦り合わせる作業こそ、対面でないと成立しない。
>  これは方法論の話で、議題の確証度とは独立」

**具体アジェンダ4点セット**：
1. **観測ファクトの共通認識化**（分母を揃える）
2. **解釈フレームの並列陳列**（各々のviewを可視化）
3. **検証設計**（何が分かれば確証できるか）
4. **不確定を抱えたままの戦略判断**（待てない意思決定との接続）

**心理ハードルを下げる枕詞**：
> 「60分で結論まで出る必要はない。各々の解釈の差分が見えるだけで価値がある」

**さらにブロックされた場合の二段返し**：
「検証設計も結局やってから議論すべきでは？」と来たら：
> 「検証設計こそ対面でやりたい。何を観察すれば確証できるかは、各々の事業観・優先順位観が出る議題で、文書では擦り合わない」

**Why**: 2026年4月29日、蔵開きPoC振り返りMTGの再戦設定をする際、「ベロ族確証性が低いから議論できない」という想定ブロックへの対応を整備した。「データ不足→話せない」は一見ロジカルだが、議論回避の常套句でもある。確定情報は非同期、不確定情報こそ同期で擦り合わせる、という方法論的原則を崩さないことがCEOの戦略議題進行に必要。

**How to apply**:
- アリーが対面MTGの議題を主導する立場で、相手から「データ不足だから話す内容ない」と来たとき
- 特に宮城信さん（構造化重視・確定志向）に対する切り返しとして有効
- 戦略仮説の修正議論や、PoC評価の対面ミーティングで使う
- 「対面MTGのアジェンダ設計原則」と併用（feedback_offline_meeting_agenda.md）
