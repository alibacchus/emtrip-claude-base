# スライド設計の原則

## 鉄則
- AIに「作らせる」のではなく「埋めさせる」
- 先に「設計テンプレート」を作ってからClaudeに渡す
- 見た目テンプレートより設計テンプレートが先

## 設計テンプレートの作り方（3ステップ）
1. 各ページの「主張」を1行で書く
2. 各ページに「だから何？」を追記する
3. Claudeには「このテンプレートを埋めて」と指示する

## EmTrip標準スライド構成

### 投資家向けデッキ（8枚構成・汎用）
1. タイトル：会社名・タグライン・日付
2. 課題：旅が消費で終わる問題（数字付き）
3. 解決策：オーナーAIによる文化共創体験
4. プロダクト：ココロクロス・マツリクロスの概要
5. トラクション：ユーザー数・エンゲージメント・店舗数
6. ビジネスモデル：B側 AI Agent + Value Share Hybrid・C側 旅後Premium・地域還元15%（C+D収益から）
7. 今後の計画：3ヶ月・6ヶ月・1年のマイルストーン
8. 資金調達：調達額・使途・チーム

### v3.6 Layer 2（日本シードVC・エンジェル・CVC）向け推奨構成（10枚・2026-05-08）

v3.6 §32 / 付録Aベース。AGI/ASIは出さない・「観光は手段、データは資産、AIは output」を貫く。

1. **タイトル**：EmTrip — Cultural Intelligence Infrastructure / Mission：みんなの思い出をみんなの価値に変える
2. **What we do**：地域住民・職人・青年会の一次ナラティブをAIインターフェース化・旅前旅中旅後の対話から文化の意味データを蓄積する文化共創SaaS
3. **Why now**：次世代AIの構造的データギャップ + 既存モデル企業の著作権訴訟・地政学リスク + 観光のconsent-clean性
4. **Production-Grade Dual-Channel Architecture**：LINE版（日本人国内・全フェーズ・本番運用）+ Web版（多言語インバウンド・全フェーズ・機能パリティ達成中）
5. **5 Tracks**：Track 1 (B Revenue) / Track 2 (C Scale) / Track 3 (D Licensing) / Track 4 (C Premium) / Track 5 (UGC Supply)

7. **Current Traction**：3店舗契約済（八女・石垣・瀬長島）・LINE版本番運用 + Web版7言語multi-tenant・PoC学習pivot確定

10. **Mid-term growth lever**：LINE版多言語化（30-50日工数）→ 台湾・タイ市場（Series Seed後）

### Layer 1（公庫・地銀・自治体）向け推奨（5-6枚）

- 「文化観光の意味データを蓄積するSaaS」「地域還元15%」を強調
- 5 Tracks や AGI/ASI は出さない
- 沖縄・八女の地域貢献・雇用・文化保全の文脈を冒頭に置く
- 既受領 JFC ¥500万の利用実績を1枚で示す

### Layer 3（米VC・deep-tech VC・AI Lab）向け推奨（英語10-12枚）

- 全英語スライド・"Cultural narrative data infrastructure for next-gen AI" を冒頭
- AGI/ASI calibrated thesis は明記（overstate を避けつつ構造的データギャップを語る）
- D層市場（AI Lab・travel platform・政府）の TAM 算出を入れる
- 比較対象は Reddit→Google ($60M/yr) / News Corp→OpenAI ($250M / 5y) など実例

### 決算サマリー（5枚構成）
1. 期間サマリー：売上・費用・損益
2. キャッシュフロー：月次推移
3. KPI：ユーザー数・CVR・リテンション
4. 課題と対策
5. 次期計画

### 事業者向け提案書（4枚構成）
1. ココロクロスとは：一言説明＋効果
2. 導入事例：他店舗の数字
3. 費用対効果：ライセンス料vs期待LTV向上
4. 導入ステップ：オンボーディング3ステップ
