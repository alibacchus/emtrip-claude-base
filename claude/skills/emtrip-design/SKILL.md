# EmTrip デザインシステム

このスキルは「EmTripのブランド骨格」と「~/Desktop/design/ の8冊から抽出した実務ルール」を統合したものである。
デザイン・LP・スライド・ポップ・マーケコピー・営業資料・投資家デッキを作るときに必ず参照すること。

---

## 1. ブランド骨格（変更厳禁）

### コーポレートカラー
- Primary：`#4653A2`（青紫）→ 見出し・主要見出しライン・メインCTA
- Secondary：`#ED6D46`（オレンジ）→ 強調ワード・サブCTA・数字ハイライト
- Background：`#FDFCF9`（ベージュ）→ 白背景代わりの紙っぽい土台
- Text：`#1A1A1A`

### タイポグラフィ
- 日本語：Noto Sans JP
- 英語：Inter
- 見出し：font-weight 700 ／ 本文：400
- line-height：日本語 1.7 ／ 英語 1.5
- フォントは「種類を増やす」のではなく「ウェイトで表情をつける」（同ファミリーでWeight 400/500/700を使い分け）

---

## 2. 禁止パターン（AIっぽさ・素人っぽさの除去）

### 絶対にやらないこと
- グラデーション多用（1画面に2箇所まで）
- box-shadowを重ねて"浮かせる"表現
- 絵文字をUIに混ぜる
- コーポレートカラー以外の派手な色を追加する
- 余白を潰して詰め込む
- ChatGPTで生成した汎用イラスト・ストック素材の多用
- シャドーを重ねがけして「立体感」を出す（『デザイン力の基本』第1章：非デザイナー失敗パターンの代表例）

### 非デザイナーが陥る5つの失敗（『デザイン力の基本』より原文ベース）
このチェックは制作後に毎回走らせる：
1. 見えない・読めない（文字が小さい／コントラスト不足／シャドー過多）
2. さえない・映えない（情報が詰め込まれ全体がごちゃごちゃ）
3. 信頼性がない・うさんくさい（色数過多・フォント不統一）
4. 存在感がない・記憶に残らない（目立たせるために大きくするだけ）
5. シェアする気にもならない（統一感の欠落）

## 3. 4つの基本原則（『ノンデザイナーズ・デザインブック』）

非デザイナーが最初に守る4原則。すべての制作物で順に確認する。

### 近接（Proximity）
- 関連する要素は物理的に近づけ、グループ間は余白で分ける
- 「何と何が1つのかたまりか」を視覚だけで伝わる状態にする

### 整列（Alignment）
- 左揃え・中央揃え・右揃えのいずれかに統一し、文字の左端を揃える
- 「揃っているようで揃っていない」を許さない

### 反復（Repetition）
- 同じ要素（色・フォント・線・アイコン・余白幅）を意図的に繰り返す
- ブランドカラー3色の役割は全制作物で統一：#4653A2=見出し／#ED6D46=CTA・強調／#FDFCF9=土台

### コントラスト（Contrast）
- 「微妙な違い」ではなく「明らかな違い」を作る
- サイズ・太さ・色を迷ったら「思い切って差をつける」

---

## 4. AISUS（『デザイン力の基本』）

EmTripが制作物に仕込むべき5つの効果。特に S と S を外さない。
- **A**ccessibility：見やすさ・読みやすさ（可読性第一）
- **I**mpression：印象・映え
- **S**incerity：信頼性・誠実さ ← **投資家資料・事業者営業資料で最重要**
- **U**niqueness：独自性（EmTripらしさ・沖縄発の手触り）
- **S**hare：共感・共創・シェアされる力 ← **SNS・LP・ポップで最重要**

---

## 5. レイアウト原則

- 余白を十分に取る（padding最小24px）
- モバイルファースト
- 情報密度は低め・読みやすさ優先
- セクションの区切りは色でなく**余白**で表現（反復の原則）
- **ワンキャッチ・ワンビジュアル**：1スライド・1セクションに訴えるメッセージは1つだけ
- 「目に見える順番」と「情報の優劣」を一致させる

### コンポーネント
- ボタン：角丸8px／Primary色背景／白テキスト
- カード：白背景／border-radius 12px／subtle shadow（Y軸 2-4px、blur 8px以内）
- ヘッダー：白背景／Primary色アクセントライン

## 6. スライド・投資家資料・事業計画書

- 1スライド1メッセージ
- テキストは最小限（箇条書き3点以内）
- 図解・数字を優先
- フォントサイズ：タイトル32px以上／本文18px以上
- 最初のスライドで「Sincerity（誠実さ）」を伝える（華美なアニメーション禁止）
- 投資家資料は `emtrip-slide-template` スキルも併用

### スライド作成チェックリスト
- [ ] 誰が見るのか（投資家／事業者／パートナー）具体的に想定した
- [ ] 1スライドで伝えたいことは1つだけ
- [ ] 数字・重要ワードはコントラストで目立たせた
- [ ] フォント・色・余白は全スライドで統一（反復）
- [ ] 箇条書きは3点以内

## 7. LP・ポップ・販促物

### 制作前に必ず決めること（『デザイン力の基本』第1章「いきなり手を動かさない」）
1. 誰に見せるのか（ペルソナ1人）
2. 何を伝えたいのか（1文で）
3. 見た人にどう動いてほしいのか（1つの行動）
4. トーン&マナー（クール／親しみ／誠実／祝祭的 のどれか1つ）

### LP構成の骨格（マーケ書籍からの統合）
- ヘッドライン（ベネフィット1文）
- 問題提起（相手の「飢え」＝ニーズ顕在化）
- 解決策の提示
- 社会的証明（導入店舗・成功事例）
- リスク逆転（無料トライアル・返金保証等）
- CTA（1つだけ）

## 7.5 2QR並列POP仕様（v3.6 §20 / 付録E・2026-05-08確定）

ココロクロス3店舗・古民家宿・観光案内所等で配備する2QR並列POP/カードの正規仕様。LINE QR（日本人国内向け）+ Web QR（多言語インバウンド向け）の併記が原則。

### POPレイアウト

```
┌────────────────┬────────────────┐
│  日本人の方     │ International   │
│   [LINE QR]    │   [Web QR]      │
│ 友だち追加で    │ Tap to chat     │
│ チャット開始    │ in 7 languages  │
└────────────────┴────────────────┘
```

### デザイン制約

- **Color**：Primary `#4653A2` / Accent `#ED6D46`（§1ブランド骨格と統一）
- **Font**：Noto Sans JP / DM Sans
- **QRサイズ**：両側統一・25mm × 25mm 推奨
- **余白**：QR周辺 6mm 以上
- **色数**：3色以内
- **AISUS**：Sincerity（誠実さ）優先・装飾は最小限

### 配置先別ルール（v3.6 §22）

| 配置先 | LINE QR | Web QR |
|---|---|---|
| 店舗店頭・蔵見学（髙橋商店・池原酒造・シーサー注意） | ◎ | ◎（並列） |
| 古民家宿 3軒（八女）・抹茶カフェ 5軒 | ◎ | ◎（並列） |
| 観光協会案内所（八女・石垣・那覇空港） | ◎ | ◎（並列） |
| ホテル群（石垣島3-5軒・那覇主要3軒・瀬長島） | ✗ | ◎（単独） |
| 瀬長島ウミカジテラス内テナント連携 | ✗ | ◎（単独） |

### オンライン出面（QR vs URL）

- **日本語コンテンツ → LINE QR 主体**：EmTrip Web 日本語SEO記事・Instagram日本語・X日本語・note・YouTube国内向け
- **多言語コンテンツ → Web QR 主体**：多言語SEO・Instagram多言語・Reddit・Custom GPT・YouTube多言語向け
- **両方併記する場面**：bilingual SNS投稿・国内+海外混在YouTuberコラボ・国内空港案内

### 制作チェックリスト

- [ ] QR両側のサイズが揃っているか（25mm × 25mm 推奨）
- [ ] QR周辺余白 6mm 以上
- [ ] 「日本人の方 / International」のラベル明示
- [ ] LINE QRは LINE登録URL・Web QRは emtrip.io/[store] へ正しくリンク
- [ ] 色は3色以内（Primary・Accent・地色）
- [ ] 「中川社長とAIで話せる」等の景表法注記が必要な場合は併記（emtrip-legal-check 参照）

## 8. マーケティング・コピーの原則

ここは ~/Desktop/design/ の5冊（HIGH POWER MARKETING／売り方が9割／説得の心理技術／たった一人の分析／成功する企業アプリ）から抽出した原文ベースの原則を運用形に整えたもの。

### 8.1 成長の3軸（『HIGH POWER MARKETING』ジェイ・エイブラハム）
売上はこの3つの掛け算でしか増えない：
1. クライアント数を増やす
2. クライアント1人あたりの平均販売額を増やす
3. クライアントの購入頻度を増やす

施策を考えるときは「どの軸を動かしているか」を必ず1つに定める。

### 8.2 USP（Unique Selling Proposition）
- 「〇〇屋さん」ではなく「顧客の〇〇を実現する存在」と定義し直す
- EmTripの例：
  - ココロクロス = 「店舗常設AI」ではなく「店主の哲学を来店客に手渡す装置」
  - マツリクロス = 「エイサー体験予約」ではなく「青年会と旅人の共創の場」
  - OPePe = 「予約管理ツール」ではなく「マリン事業者の現場負担を半分にする相棒」

### 8.3 リスク逆転（Risk Reversal）
不況・資金制約下で最も効く販促：購入前のリスクを「企業側が背負う」宣言をする。
- 「成果が出なければ全額返金」
- 「3ヶ月無料トライアル」
- 「導入初月の地域還元15%は初回分無料」
導入ハードルが下がり、実績が薄い段階のEmTripに特に効く。

### 8.4 顧客が離反する理由（『売り方が9割』）
顧客が離れる理由のほとんどは「品質不満」でも「価格」でもなく「**あなたの存在を忘れている**」。
→ 定期接触（月1以上のメール・SNS・LINE・イベント通知）がサブスク継続率の根っこ。

### 8.5 説得の心理技術（Dave Lakhani）
操作（manipulation）ではなく説得（persuasion）でいく。6つの心理レバー：
1. **社会的同調**（Social Proof）：導入店舗の声・他店の成功事例
2. **譲歩／返報性**（Reciprocity）：無料オーナー相談・無償テンプレ配布を先に出す
3. **権威**（Authority）：沖縄・青年会・一次文化保持者との連携・実績を可視化
4. **希少性**（Scarcity）：「PSF検証中のため先行3店舗限定」等
5. **親近感**（Familiarity）：CEOアリーの「非デザイナーCEOらしさ」を素材にする
6. **排他性**（Exclusivity）：導入オーナーのみのクローズドコミュニティ

### 8.6 顧客起点マーケティング（西口一希『たった一人の分析から…』）
**N1分析**：実在する1人の顧客（名前・顔が思い浮かぶ人）を深掘りする。
- 「なぜあなたはEmTripを使い続けるのか」をそのまま本人の言葉で記録する
- ロイヤル化の「きっかけ」を特定し、未ロイヤル層に同じきっかけを提供する
- **9セグマップ**：購買頻度 × ブランド選好 の2軸で顧客を9セグに分類し、セグ別施策を設計

### 8.7 ストーリーテリングの3層（『説得の心理技術』）
ピッチ・LP・note記事で必ず3本用意する：
1. **自分たちのストーリー（Why）**：なぜEmTripをやっているか
2. **顧客のストーリー（How）**：導入オーナーの変化物語
3. **業界・地域のストーリー（What）**：沖縄・地域・文化の置かれた構造

### 11.1 『AI時代のYouTuber 成功法則』（植木信詞）— コンテンツ設計・集客

**原文で確認した中核概念**：
- YouTubeは日本国内だけで月間7000万人が訪れる巨大マーケットで、世界全体で1日10億時間以上視聴されている
- 本書は「顔出ししないYouTube」を中心テーマとする。外注化・仕組み化により、特殊能力がなく副業時間が限られる人でも成果を出せる前提で構成
- リサーチ・企画選定・データ分析の重要性が強調される
- 「感情的にならずに高速で改善していく」運用姿勢を基本とする
- ジャンル選定・コンセプト選定・台本・タイトル＆サムネイル・動画作成・アナリティクスの順で設計
- 登録者数のフェーズごとに戦い方を変える必要がある
- 「AI導入による社会の未来は暗い」と警鐘を鳴らしつつ、AIとYouTubeを組み合わせた未来についても扱う

**EmTripでの適用ルール（推論含む）**：
- **推論：** EmTrip公式アカウント（ココロクロス・マツリクロス・OPePe）で動画コンテンツを運用するとき、「顔出ししない／仕組み化／外注化」の前提でLTVを設計する
- リサーチ → 企画 → 台本 → タイトル・サムネ → 動画 → アナリティクス分析のフローを守る。いきなり撮影から始めない
- 各コンテンツは「感情でなく数字」で改善判断する。再生・維持率・CTRの閾値を事前に決めてから公開
- 登録者数フェーズ（0→1000／1000→10k／10k→）で戦い方を変える。ショート中心／長尺中心を数字で切り替え
- 動画サムネは §5 のブランドカラー・フォント原則に加え、「冒頭3秒で結論が見える」構成を守る（YouTubeアルゴリズム特性）

### 11.2 『おばあちゃんがWEBマーケティングを勉強して雑貨屋をV字回復！』— 非専門家向けマーケ基礎

**原文で確認した中核概念**：
- Webマーケティングとは「顧客を想像し市場を創造すること」。広告だけでなく、徹底的な消費者視点で価値を届ける活動全般
- Webマーケティングのプロセスは **集客 → 行動 → CV → リピート** の4段階で、各プロセスをデータで横ぐし分析しPDCAを回す
- 集客の手段：**SEO／広告／SNS／メール** の4経路
- SEOは「お金をかけずに見込み客を自分のサイトに連れてくる」ため最重要スキル
- リピート構造を作る（ファンづくり）＋UX（ユーザーエクスペリエンス）検証によりCVに繋げる、が全体ストーリー
- 本書は小説形式で、非専門家（73歳のおばあちゃん）が孫のマーケター青年の指導で学ぶ構成

**EmTripでの適用ルール（推論含む）**：
- **推論：** 非デザイナー・非マーケターであるアリー自身が施策を打つ前に、集客 → 行動 → CV → リピートの4プロセスのどこを触っているかを明示する
- 単発の広告キャンペーンでなく、SEO（オーナーノート・note記事）を無料の見込み客集客チャネルとして最重要視する
- LP・オウンドメディア改修のたびに「このページはどのプロセス（集客／行動／CV／リピート）を担うか」を先に決める
- Webマーケの範囲を「広告運用」に矮小化せず、「顧客を想像し市場を創造する」全体活動として位置づける。§8 のコミュニケーション戦略とペアで使う
- オーナー向けサポートでも、この4プロセスで現状を整理して伝える（専門用語を避け、ストーリー形式で示す）
