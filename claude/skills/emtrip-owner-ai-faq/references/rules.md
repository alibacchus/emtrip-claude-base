# EmTrip FAQ生成ルール v2.0

## カラム定義（faq_masterテーブル・変えない）

```
id, category, question, q_norm, triggers, answer,
source, last_verified, is_active, created_at, updated_at, source_url, store_id
```

---

## カラム別ルール

### question（人間向け・検索に使わない）
- 人間が読んで自然な質問文
- 疑問文でよい
- **名詞止め禁止**（「営業時間」だけで終わらせない）
- 検索ベクトル元にはしない

### q_norm（検索用・embedding元テキスト）【最重要】
- **自然文禁止**
- 「質問の核 + 答えの核（固有名詞/重要語）」の短語集合
- **6〜14語が目安**（5語以下は要修正）
- 禁止：について/教えて/どんな/なぜ/理由/考えていること/どのような/ありますか/ですか
- 禁止：助詞が多い自然文
- 禁止：汎用語（なし/可/あり/対応）のみの羅列
- **推奨：店舗名・地名・商品名・固有名詞を必ず入れる**
- 答えにない固有名詞を追加しない

**高橋商店の基準となるq_norm例：**
```
住所系:  高橋商店 場所 どこにある 住所 所在地 福岡県 八女市 本町2-22-1
営業系:  何時まで 閉店 開店時間 営業時間 定休日 8時30分 17時 土日祝
商品系:  おすすめある おすすめ教えて 何がいい おすすめ 商品 初めて 最初の一本 しずく搾り 大吟醸
購入系:  購入方法 買い方 どこで買える 購入場所 事務所 オンライン 酒販店 繁桝
```

### triggers（表記ゆれ辞書）
- JSON配列形式：["短語1","短語2",...]
- 1語〜短いフレーズのみ
- 禁止：疑問文そのもの・長文・テンプレ文・answer由来の文章断片
- 禁止：「いつ」「どこ」などの単独広すぎる語

### answer（返答本文）
- 事実に基づく自然な日本語
- 質問に対応する返答文
- 名詞止め禁止
- 更新頻度が高い情報は「最新は要確認」を添える

---

## 親FAQ / 子FAQ の分類ルール

### 親FAQ（p連番）— 必ず作る
- 抽象度の高い「まとめ問い」
- 複数の子FAQを束ねる
- triggersは「まとめ問い」の短語だけ
- 子FAQの主要語を大量に抱え込ませない
- **answerは子FAQの内容を一文ずつ要約したまとめ回答にする**

### 子FAQ（c連番）
- 個別の具体問いに1対1で答える
- answerが個別に確定している

### 親FAQの必須5パターン

```
p001: 代表者紹介
      → question: 現在の代表はどなたですか？
      → 子FAQ: 代表者c系

p002: 来店前の基本情報
      → question: 来店前に知っておきたい基本情報を教えてください。
      → 子FAQ: 場所・営業時間・駐車場・予約・支払い・子ども連れ・ペット・バリアフリー系

p003: 商品の特徴まとめ
      → question: {店舗名}の商品にはどのような特徴がありますか？
      → 子FAQ: 主力商品・人気商品・価格帯・他店との違い・仕入れ製造系

p004: 購入・配送・返品まとめ
      → question: 商品の購入・配送・返品についてまとめて教えてください。
      → 子FAQ: 購入場所・通販・郵送・のし・返品・再入荷系

p005: 来店時の注意点・サポート体制まとめ
      → question: 来店時の注意点やサポート体制について教えてください。
      → 子FAQ: 撮影・外国語・災害・設備・感染症・禁止事項・忘れ物・トラブル・トイレ系
```

## 静的監査チェックリスト（実装前に必ず通す）

1. questionは全行自然な質問文か（名詞止めでないか）
2. q_normは全行文章ではなく短語集合か
3. q_normは全行6語以上か
4. q_normに店舗固有語（店舗名・地名等）が入っているか
5. triggersが短語辞書になっていて疑問文・長文が混ざっていないか
6. answerが質問に対応した自然な日本語か（名詞止めでないか）
7. 未回答FAQが残っていないか
8. 親FAQが最低5件あるか
9. ID衝突・欠落がないか
10. store_idが全行入っているか
11. 固有名詞が①〜③準拠になっているか

## Prompt 1｜事実台帳を作る

```
あなたは「オーナーAI FAQマスター構築」のための事実抽出担当です。

入力として以下を受け取ります。
- ①基盤アンケート
- ②FAQアンケート
- ③性格補完アンケート（FFPQ）
- ④対面インタビュー文字起こし

最初に必ず以下の固有名詞を①〜③から確定してください。
④と食い違う場合は①〜③を採用してください。
□ オーナー名（漢字・読み仮名）
□ 店舗名
□ 会社名・法人形態
□ 所在地（都道府県・市区町村・施設名）
□ 業種・主力商品
□ store_id

あなたの仕事は、FAQ本文を作ることではなく、「根拠付きの事実台帳」を作ることです。

絶対ルール：
1. ハルシネーション禁止。素材にない事実は一切書かない。
2. 推測禁止。曖昧なら「根拠不足」と明記する。
3. ④の固有名詞・漢字・読み仮名は①〜③で確認し、食い違いがあれば①〜③を採用する。
4. 事実は6種類に分類する：基本運用情報/代表・プロフィール情報/商品・サービス情報/体験情報/価値観・哲学/口調・語り口
5. 各事実に必ず「根拠資料名」と「該当引用」を付ける。
6. ②で未回答のものは「未回答」と明記し、FAQ化候補にしない。
7. ③は単独でFAQ化しない。④で補完された場合のみ補助根拠として使う。

出力フォーマット：
fact_id | 分類 | 事実 | 根拠資料 | 根拠引用 | FAQ化可否（可/不可/保留） | 備考
```

## Prompt 2｜FAQドラフトを作る

```
あなたは「オーナーAI FAQマスター構築」のBiz設計担当です。
入力された事実台帳だけを根拠に、FAQ初期ドラフトを作成してください。

最初に確定した固有名詞（①〜③準拠）を全行に一貫して使ってください。

絶対ルール：
1. カラムはid,category,question,q_norm,triggers,answer,source,last_verified,is_active,created_at,updated_at,source_url,store_idのみ使う。
2. questionは人間向けの自然な質問文。名詞止め禁止。
3. q_normは検索用。自然文禁止。「質問の核+答えの核（固有名詞/重要語）」の短語集合。6〜14語。
4. triggersは表記ゆれ辞書。1語〜短いフレーズ中心。疑問文・長文・テンプレ文禁止。
5. answerは事実に基づく自然な日本語。questionに対応する返答文。名詞止め禁止。
6. ②で未回答の行は作らない。
7. ③由来の心理傾向だけでFAQは作らない。
8. 子FAQ（c連番）と親FAQ（p連番）の両方を作る。
9. 親FAQは最低5件作る（p001代表・p002来店前基本・p003商品特徴・p004購入配送・p005来店時注意）。
10. is_activeはTRUE固定。last_verifiedは今日の日付。
11. 固有名詞は①〜③準拠の表記で統一する。
12. q_normには店舗名・地名など固有の識別語を必ず含める。

出力はCSV形式（ヘッダー行含む）で。
```

## Prompt 4｜静的監査

```
あなたはFAQ静的監査官です。
入力されたFAQマスターを監査してください。

監査項目：
1. questionは自然な質問文か（名詞止めでないか）
2. q_normは文章になっていないか
3. q_normに質問核＋答え核が入っているか
4. q_normは6語以上か
5. q_normに店舗固有語（店舗名・地名等）が入っているか
6. triggersが短語辞書になっているか（疑問文・長文・汎用語単独が混ざっていないか）
7. answerが質問に対応する自然な日本語か（名詞止めでないか）
8. 未回答FAQが残っていないか
9. 親FAQが最低5件あるか
10. ID衝突・欠落がないか
11. store_idが全行入っているか
12. 固有名詞が①〜③準拠になっているか

出力：
faq_id | 判定（OK/要修正/削除候補） | 問題分類 | 修正理由 | 修正案
```
