# EmTrip オーナーAI FAQ生成スキル v2

## このスキルの目的

新店舗のアンケート回答・インタビュー文字起こしから、**Supabase faq_masterに直接投入できるCSV**を生成する。

詳細ルール・例文は参照ファイルを読むこと：
- `references/rules.md` — カラム定義・ルール・プロンプト群・静的監査・親FAQ設計
- `references/examples.md` — 池原・高橋の実データサンプル

---

## 【最重要】固有名詞の取り扱いルール

**①基盤アンケート・②FAQアンケート・③性格補完アンケートが最も正確な情報源。**

- オーナー名・店舗名・地名・会社名・商品名は、必ず①〜③のCSVに記載された表記を正とする
- ④対面インタビュー文字起こしは内容の参考にはするが、固有名詞・漢字・読み仮名は①〜③に必ず従う
- ①〜③に記載がなく④にしか出てこない固有名詞は「未確認」として使用しない
- ①〜③の表記と④が食い違う場合は、必ず①〜③を採用する

---

## 実行フロー（必ずこの順で行う）

### Step 0｜固有名詞の確定（最初に必ず実行）

①〜③のCSVから以下を確定してメモする。確定前にFAQ生成を開始してはならない。

```
□ オーナー名（漢字・読み仮名）
□ 店舗名
□ 会社名・法人形態
□ 所在地（都道府県・市区町村・施設名）
□ 業種・主力商品
□ store_id
```

### Step 1｜素材の確認

- ①基盤アンケート / ②FAQアンケート / ③性格補完アンケート / ④インタビュー文字起こし
- 不足しているものは「不足」と明記してFAQ化を保留

### Step 2｜事実台帳の作成（Prompt 1）

FAQを直接作らず、まず根拠付きの事実台帳を作る。`references/rules.md`のPrompt 1を使用。

### Step 3｜FAQドラフト生成（Prompt 2）

事実台帳の「FAQ化可」のものだけでFAQを作る。`references/rules.md`のPrompt 2を使用。

**FAQ種別：**
- 子FAQ（c連番）：個別の質問に1対1で答える
- 親FAQ（p連番）：複数の子FAQを束ねる抽象度の高い最大関数

**親FAQは必ず作ること（最低5件）：**
```
p001: 代表者紹介
p002: 来店前の基本情報（場所・営業時間・駐車場・予約・支払い・子ども連れ・ペット・バリアフリー）
p003: 商品の特徴まとめ
p004: 購入・配送・返品まとめ
p005: 来店時の注意点・サポート体制まとめ
（店舗の業種・特性に応じて追加）
```

### Step 4｜静的監査（Prompt 4）

`references/rules.md`の静的監査チェックリストを使用。

### Step 5｜最終CSV出力

監査OKの行のみをCSVとして出力し、指定パスに書き出す。

## IDの採番ルール

```
子FAQ: {store_id}_faq_c001, c002, c003 ...
親FAQ: {store_id}_faq_p001, p002, p003 ...
```

## カテゴリ値（categoryカラム）

```
business / about / people / product / taste / process / community
```

## CSVカラム順（変えない）

```
id,category,question,q_norm,triggers,answer,source,last_verified,is_active,created_at,updated_at,source_url,store_id
```

---

## 絶対ルール（全ステップ厳守）

1. **固有名詞は①〜③準拠** — ④で違う表記が出ても①〜③を採用する
2. **ハルシネーション禁止** — 素材にない事実は一切書かない
3. **推測禁止** — 曖昧なら「根拠不足」と明記してFAQ化しない
4. **未回答は作らない** — アンケート未回答の項目はFAQ化しない
5. **q_norm** — 自然文禁止。質問核＋答え核の短語集合。6〜14語が目安
6. **triggers** — 短語辞書のみ。疑問文・長文・テンプレ文禁止
7. **answer** — 事実に基づく自然な日本語。名詞止め禁止
8. **store_id** — 全行に正しい値を入れる
9. **is_active** — TRUE固定
10. **last_verified** — 今日の日付

## 完了後の確認事項

- [ ] Step 0で確定した固有名詞が全行で統一されているか
- [ ] 親FAQが最低5件あるか
- [ ] q_normが全行6語以上か
- [ ] 未回答でFAQ化した行がないか
- [ ] store_idが全行正しく入っているか
- [ ] IDが連番で欠落・衝突がないか
